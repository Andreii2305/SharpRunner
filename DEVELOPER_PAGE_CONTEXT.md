# Developer Page Context (Checkpoint)

Last updated: 2026-03-24

## Current Direction

- Developer tools are now separated from normal user/admin login flow.
- Admin account creation can be controlled through one-time invite codes generated by a developer session.
- Current implementation is MVP-first: manual invite generation and manual sharing.

## What We Completed

- Added developer-only API login using `DEVELOPER_SETUP_KEY`.
- Added short-lived developer JWT session (scope: `developer`).
- Added admin invite persistence model (`AdminInvites`).
- Added invite management APIs:
  - generate invite code
  - list recent invites
  - count active invites
- Added admin registration via invite code:
  - validates code existence
  - checks expiration
  - checks one-time use
  - optional invited-email restriction
- Added frontend pages:
  - `/developer` for generating and viewing invite codes
  - `/admin-invite` for admin self-registration using invite code

## Current API Contract (Developer Flow)

- `POST /api/developer/login`
  - Body: `{ "setupKey": "<DEVELOPER_SETUP_KEY>" }`
  - Returns developer token (8h expiry)
- `POST /api/developer/admin-invites`
  - Headers: `Authorization: Bearer <developer-token>`
  - Body:
    - optional `invitedEmail`
    - optional `expiresInHours` (default 72)
- `GET /api/developer/admin-invites`
  - Headers: developer token
  - Returns recent invite rows
- `GET /api/developer/admin-invites/active/count`
  - Headers: developer token
- `POST /api/auth/register-admin-invite`
  - Body: `inviteCode`, `firstName`, `lastName`, `username`, `email`, `password`
